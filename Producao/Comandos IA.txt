Ao receber um comando abreviado realize a sequência de comandos descritas.

- Comando: commit
- Ação: Adiciona todos os arquivos alterados, faz commit com mensagem padrão e realiza push para o GitHub.
- Sequência de comandos:
  - git add .
  - git commit -m "<<comentário>>" (<<comentário>> deve ser a descrição do que foi atualizado no sistema)
  - git push

- Comando: push <arquivo>
- Ação: Adiciona, faz commit e push de um arquivo específico.
- Sequência de comandos:
  - git add <arquivo>
  - git commit -m "Atualização de <arquivo> pelo Agent"
  - git push

- Comando: dpl-atend
- Ação: Realiza push e deploy do módulo Atendimento no Apps Script.
- Sequência de comandos:
  cd Atendimento ; clasp push ; clasp deploy --deploymentId AKfycbyjHfu_JThCljCgGYNOMn38cO4OwBNcBgEiWlQnMMLV29uao5FSsSTXUr-kc1B8ed9j --description "<<comentário>>" (<<comentário>> deve ser a descrição do que foi atualizado no módulo Atendimento) ; cd..

- Comando: dpl-cid
- Ação: Realiza push e deploy do módulo Cidadao no Apps Script.
- Sequência de comandos:
  cd Cidadao ; clasp push ; clasp deploy --deploymentId AKfycbxvkce95ZEE84wqed5ltl1ZgkHyt4CGyPzMiq-zHJfXkHyL01X70xWU0Ot14scMd3sW --description "<<comentário>>" (<<comentário>> deve ser a descrição do que foi atualizado no módulo Cidadão) ; cd..

- Comando: dpl-email
- Ação: Realiza push e deploy do módulo Email no Apps Script.
- Sequência de comandos:
  cd Email ; clasp push ; clasp deploy --deploymentId AKfycbzVUDExZbAyLYVQ-8CAbAg3JjKA3cQ1NVv60P3-c9F2HC8Gtvkr4wb1uxjgrf65NZF7 --description "<<comentário>>" (<<comentário>> deve ser a descrição do que foi atualizado no módulo Email) ; cd..

- Comando: dpl-all
- Ação: Realiza commit, push e deploy de todos os módulos (Atendimento, Cidadao e Email) em sequência.
- Sequência de comandos:
  - git add .
  - git commit -m "<<comentário>>" (<<comentário>> deve ser a descrição do que foi atualizado no sistema)
  - git push
  - cd Atendimento
  - clasp push
  - clasp deploy --deploymentId AKfycbxvkce95ZEE84wqed5ltl1ZgkHyt4CGyPzMiq-zHJfXkHyL01X70xWU0Ot14scMd3sW --description "<<comentário>>"
  - cd ../Cidadao
  - clasp push
  - clasp deploy --deploymentId AKfycbzUiUkAP9XQ3gCo0vOHswNt78jV-SJpx_RulNzgDh6G680XTx8VEA52VA_CdyDd86erGg --description "<<comentário>>"
  - cd ../Email
  - clasp push
  - clasp deploy --deploymentId AKfycbzVUDExZbAyLYVQ-8CAbAg3JjKA3cQ1NVv60P3-c9F2HC8Gtvkr4wb1uxjgrf65NZF7 --description "<<comentário>>"
  - cd ..

Após finalização de cada comando, registre no arquivo Atualizacoes.txt existente na pasta do módulo que está sendo atualizado o número da ação (se o arquivo estiver em branco inicie com a #1), o problema apresentado, qual foi a solução proposta e, após a atulização dos códigos, fale para o programador testar. Se a resposta for não funcionou, registre que a ação falhou e aguarde próximo comando. Se a resposta for funcionou, registre que a ação funcionou. Se a resposta for funcionou parcialmente, registre a resposta do que funcionou e o que falhou na ação e feche o registro da ação.
